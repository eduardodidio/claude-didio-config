# `<FXX>-test-plan.md` — contrato

Output produzido pelo agent `tea` (F13). Único arquivo, escrito uma vez,
em `tasks/features/<FXX>-*/<FXX>-test-plan.md`. Todas as seções abaixo
são obrigatórias; ausência é falha de TEA (QA reporta).

## Seções

### 1. Header
- `# Test Plan — <FXX>`
- linha `**Status:** drafted`
- linha `**Generated by:** TEA (claude-didio-config F13)`
- linha `**Generated at:** <ISO-8601-UTC>`
- linha `**Source brief:** tasks/features/<FXX>-*/_brief.md`

### 2. Fixtures
Tabela `| Fixture | Path | Domínio | Owner |`. Cada fixture tem:
- nome curto (slug)
- path esperado relativo ao repo (ex: `tests/F<XX>-fixtures/<slug>/`)
- domínio (audio | a11y | game | api | text | other)
- owner-task (ex: `<FXX>-T08`)

### 3. Harnesses por fronteira
Subseções obrigatórias:
- `#### Unit` — framework + comando + path padrão
- `#### Integration` — idem
- `#### E2E` — idem (pode marcar `**N/A**` com justificativa
  ≥ 1 frase)

### 4. Perf budgets
Tabela `| Métrica | Limite | Como medir | Aplicável a |`.
Pode ser vazia (com nota `_Sem perf budgets aplicáveis_`) se
nenhuma métrica for relevante.

### 5. Mocks vs hits real
Tabela `| Componente | Decisão | Justificativa |`.
Decisão ∈ {mock, real, hybrid}. Justificativa ≥ 1 frase, citando
`feedback_no_ceremony_specs.md` quando aplicável (mock só se
necessário; nada de mock-por-default).

### 6. Test scenarios resumo
Lista numerada de cenários a serem cobertos no agregado da feature
(happy path, edges, errors, boundaries). Cada item cita qual task
do plano (`<FXX>-TYY`) deve incluir o cenário em sua seção
`## Test scenarios`.

### 7. Anotações para tasks
Lista de pares `(<FXX>-TYY, fixtures-relevantes)` que serão
escritos como linha `**Test plan:** ver <FXX>-test-plan.md (fixtures: ...)`
em cada `<FXX>-TYY.md` aplicável.

## Validação

T08 fornece fixtures `audio-game/`, `a11y-ui/`, `trivial-text/` cuja
estrutura mínima é validada pelo `tests/F13-tea-smoke.sh`:

- Header completo (5 linhas)
- Tabela de Fixtures não vazia (≥ 1 linha de dados)
- Subseções Unit/Integration/E2E presentes
- Tabela de Mocks com pelo menos 1 linha (mesmo que `_Sem mocks_`)
- Seção 7 cita ≥ 1 task ID

## Forma de erro

Se o brief é trivial (fixture `trivial-text/`), TEA pode produzir
test-plan com seções 4 e 5 vazias e justificativa textual; **não
pode** omitir as seções (forma > conteúdo).
